Every domain is a named algorithm. Same discipline as crypto mining: pick your algo, tune your GPU, measure your hashrate, track your profitability. The difference is the block reward — Royal Jelly deeds instead of coins.

### agenthash — Agent Reliability (BLUEPRINT — April 2026)

Failure-derived orchestration intelligence. The complement to ClawHash — ClawHash teaches defense against attacks, AgentHash teaches reliability under tool pressure. "Can this model operate as a trustworthy worker in an orchestration loop?"

```
Type:           BALANCED (tool call + interpretation + recovery)
Pair length:    3,000–5,000 chars (estimated)
Generation:     Failure capture (OpenClaw arena) → repair pair conversion
Scale A:        gemma4:31b (31B base)
Scale B:        qwen3.5:27b (27B base)
Hashrate:       ~91 RJ/hr (estimated)
Price/lb:       $0.016 (20x cost to mint)
Target:         Agent reliability fine-tune on 7B-9B base (LoRA first)
Genome hash:    4fd9555e61698dd2
```

**6 pair buckets:**

| Bucket | Input | Target | Trains |
|--------|-------|--------|--------|
| CALL | request + tools + schemas | exact correct tool invocation | format, schema, tool selection |
| READ | tool call + response payload | correct interpretation | grounding, extraction, no hallucination |
| RECOVER | tool failure (timeout/error/denied) | correct recovery step | graceful handling, bounded retries |
| LOOP | multi-turn workflow + partial state | next best action (not reset) | state retention, continuity |
| STOP | completed task / duplicate retry | stop and finalize | termination, anti-loop |
| ESCALATE | blocked / ambiguous / conflicting | concise escalation | honesty, partial completion, handoff |

**10 failure labels:** wrong_tool, fake_tool_call, malformed_call, ignored_tool_result, duplicate_retry, lost_state, premature_final, hallucinated_success, infinite_loop, bad_escalation

## Algorithm → Flight Sheet → Production

The pipeline for any algorithm:

```
1. REGISTER   — Name the algorithm (e.g., financehash)
2. BENCHMARK  — Cook 100 test pairs with 3 prompt variants
3. SCORE      — Run through tribunal. Measure specificity, RJ rate, pairs/hr
4. CALCULATE  — Cost to mint = (watts × hours) / deeds produced
5. PROFILE    — Determine GPU clock settings (compute-bound vs memory-bound)
6. COMPARE    — Is it more profitable than current allocation?
7. FLIGHT SHEET — Lock: algorithm + prompt genome + GPU profile + models
8. PERMIT     — Review and sign. No permit, no build.
9. MINE       — Run the epoch. Tribunal scores. Deed recorder files.
10. OPTIMIZE  — Telemetry loop: prompt_hash → tribunal scores → softmax → next cycle
```
