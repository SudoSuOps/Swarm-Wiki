# SafeStore

SafeStore (`data/factory/safestore.py`) implements the "never lose a pair" guarantee. Every pair generated by the factory is persisted across three independent tiers so that crashes, network failures, or partial runs never result in data loss.

## Three-Tier Persistence

### Tier 1: Local JSONL
- **Frequency**: Every pair, immediately on generation
- **Location**: Local disk JSONL file in the factory output directory
- **Latency**: Instant (fsync append)
- **Purpose**: Crash recovery baseline. If the process dies, every pair up to the crash point is on disk.

### Tier 2: R2 Shard Push
- **Frequency**: Every 500 pairs
- **Location**: Per-vertical R2 bucket (sb-cre, sb-medical, etc.)
- **Format**: `shard_NNNN.jsonl` (sequential numbering)
- **Purpose**: Durable cloud backup. Survives disk failure.

### Tier 3: Supabase Stream
- **Frequency**: Every 100 pairs
- **Location**: Supabase database table
- **Purpose**: Queryable access for analytics, filtering, and dataset assembly without downloading full shards.

## Crash Recovery

On restart, SafeStore:
1. Reads the local JSONL file (Tier 1)
2. Finds the last emitted pair ID
3. Resumes generation from that point

No pairs are regenerated or duplicated. The dedup gate (MD5 fingerprint) provides a second line of defense against any edge case where a pair might be double-emitted.

## Shard Lifecycle

```
Generate pair
  -> Append to local JSONL (Tier 1, every pair)
  -> Buffer in memory
  -> At 100 pairs: stream batch to Supabase (Tier 3)
  -> At 500 pairs: push shard_NNNN.jsonl to R2 (Tier 2)
  -> Reset buffer, increment shard counter
```

## Design Rationale

The factory runs long grinding sessions -- 100K+ deals generating ~1M pairs over hours. A single point of failure (local disk only, or cloud only) is unacceptable at this scale. The three-tier approach means:

- **Power loss**: Tier 1 has everything up to the last pair
- **Disk failure**: Tier 2 has everything up to the last 500-pair boundary
- **Need to query**: Tier 3 provides SQL access without downloading shards

Cost is negligible: R2 has no egress fees, Supabase streaming is within free tier for the insert volume.
