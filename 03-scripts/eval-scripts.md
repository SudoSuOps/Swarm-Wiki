# Eval Scripts

Evaluation suites for validating model quality across verticals. All eval sets are JSONL files with structured prompts and expected outputs.

## Eval Suites

### eval_swarmcapital.jsonl

Capital markets evaluation across 11 CRE domains with 5 quality tiers.

| Property | Value |
|----------|-------|
| Prompts | 180 |
| Domains | 11 (debt origination, equity placement, loan workout, distressed assets, capital stack, credit committee, deal packaging, waterfall, underwriting, market analysis, regulatory) |
| Tiers | 5 -- Bronze, Silver, Gold, High, Platinum |
| Purpose | Validates SwarmCapitalMarkets-27B and CRE vertical models |

Tier definitions:
- **Bronze**: Basic factual accuracy, correct terminology
- **Silver**: Structured output, proper calculations
- **Gold**: Nuanced reasoning, edge case handling
- **High**: Multi-step analysis, cross-domain synthesis
- **Platinum**: Expert-level judgment, actionable recommendations with caveats

## Deterministic Gates

Six deterministic quality gates live in `data/factory/gates.py`. These run without any LLM -- pure code checks:

1. **JSON validity** -- parseable structured output
2. **Schema completeness** -- all required fields present
3. **Enum validation** -- values within allowed sets
4. **Numeric bounds** -- calculations within realistic ranges
5. **Cross-field consistency** -- related fields agree (e.g., NOI = revenue - expenses)
6. **Format compliance** -- output matches expected template

Subjective quality assessment (tone, reasoning depth, expert credibility) is handled by SwarmCurator-27B as a task type rather than a standalone judge model. The dedicated SwarmJudge franchise was killed on 2026-03-06 -- deterministic gates plus the 27B subjective call proved simpler and cheaper.

## Running Evals

Eval scripts typically:

1. Load the eval JSONL
2. Send each prompt to the target model endpoint
3. Score responses against expected outputs or rubrics
4. Output a summary with per-category pass rates and tier distribution

```bash
# Example: eval against vLLM endpoint
python3 eval_script.py \
  --eval-file eval_swarmcapital.jsonl \
  --endpoint http://swarmrails:8082/v1/completions \
  --model swarmcurator-27b \
  --output results.jsonl
```
